# Users List - QA Prototype

Прототип для QA тестування функціональності списку користувачів згідно User Story 1.

## Опис

Цей прототип реалізує сторінку списку користувачів з можливостями пошуку, сортування та пагінації.

## Функціональність

### Основні елементи
- ✅ Заголовок сторінки "Users"
- ✅ Поле пошуку з валідацією (максимум 40 символів)
- ✅ Відображення кількості знайдених користувачів
- ✅ Таблиця користувачів з колонками: Number, First Name, Last Name, E-mail, Registration Date, View
- ✅ Пагінація з вибором кількості користувачів на сторінці (10, 20, 50, 100)

### Пошук
- Пошук за First Name, Last Name або E-mail (без урахування регістру)
- Обмеження 40 символів з валідацією
- Пошук по Enter або кліку на іконку лупи
- Кнопка очищення пошуку та скидання сортування
- Повідомлення "No users found" якщо нічого не знайдено

### Сортування
- Сортування по First Name, Last Name, E-mail
- Перший клік → за зростанням, другий → за спаданням
- За замовчуванням сортується по Registration Date (за зростанням)
- Візуальні індикатори напрямку сортування

### Пагінація
- 10 користувачів на сторінку (за замовчуванням)
- Можливість вибрати: 10, 20, 50, 100 користувачів на сторінку
- Кнопки First, Previous, Next, Last
- Відображення "Page X of Y"
- Кнопки автоматично деактивуються відповідно до стану

### Дані
- Автоматична генерація 45 тестових користувачів при першому відкритті
- Збереження даних в localStorage
- Автоматичне завантаження при відкритті сторінки

## Встановлення та запуск

1. Клонуйте репозиторій або завантажте файли
2. Відкрийте `index.html` у браузері
3. Дані автоматично згенеруються та збережуться в localStorage

## Примітки для QA

- Дані зберігаються в localStorage браузера
- Для очищення даних виконайте в консолі: `localStorage.clear()`
- View link відкривається в новій вкладці (User Story 2 буде реалізована пізніше)
