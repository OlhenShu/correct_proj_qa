# Users List - QA Prototype (Buggy Version)

**Ця версія містить навмисні баги для тестування QA.**  
Порівнюйте з коректним прототипом у батьківській папці `correct_proj_qa`.

## Як використовувати

1. Відкрийте `index.html` у браузері.
2. Дані зберігаються в `localStorage` під ключем `users_buggy` (окремо від коректної версії).
3. Для скидання даних: у консолі виконайте `localStorage.removeItem('users_buggy')` та перезавантажте сторінку.

## Примітка для QA

Не читайте список багів нижче до завершення тестування — використовуйте його лише для перевірки знайдених дефектів або підказок.

### Функціональні баги

1. **Пошук чутливий до регістру** — має бути без урахування регістру (case-insensitive).
2. **Порожній пошук** — при порожньому полі та натисканні «Search» показується 0 результатів замість усіх користувачів.
3. **Обмеження 40 символів** — поле приймає до 45 символів (`maxlength="45"`), має бути 40.
4. **Валідація 40 символів** — повідомлення: «Maximum 40 characters allowed.» замість «This field should not contain more than 40 characters.»
5. **41-й символ** — можна ввести 41-й символ (не блокується введення).
6. **Кнопка Clear** — не скидає сортування до «Registration Date (asc)».
7. **Сортування за замовчуванням** — за замовчуванням сортування за Last Name, має бути за Registration Date (asc).
8. **Сортування по First Name** — перший клік дає descending замість ascending.
9. **Колонка Number** — є сортування (клік по заголовку), за вимогами сортувати не можна.
10. **View** — посилання View відкривається в тій самій вкладці (немає `target="_blank"`).
11. **«Users per page»** — при зміні значення не відбувається перехід на 1-шу сторінку; можлива порожня сторінка.
12. **Пагінація** — після зміни «users per page» може залишатися некоректна поточна сторінка (наприклад, Page 5 of 2).

### Нефункціональні (UI/текст) баги

13. **Заголовок сторінки** — «User List» замість «Users».
14. **Заголовок h1** — розмір шрифту 24px замість 32px.
15. **Placeholder пошуку** — «Serch by» замість «Search by» (опечатка).
16. **Підпис кількості** — «Users count:» замість «Found users:».
17. **Колонка таблиці** — «Email» замість «E-mail».
18. **Кнопка пагінації** — «Prev» замість «Previous».
19. **Колір повідомлення про помилку** — зелений замість червоного.
20. **Повідомлення «No users found»** — синій колір замість сірого.
21. **«Page X of Y»** — менший шрифт (11px) і інший колір, ніж у коректній версії.

### Навмисні зависання (performance/UI responsiveness)

22. **Пошук** — при введенні 3+ символів і натисканні Search інтерфейс «зависає» ~2.5 сек.
23. **Кнопка Clear** — при натисканні інтерфейс зависає ~1.5 сек.
24. **Сортування по E-mail** — при кліку по заголовку колонки E-mail зависання ~2 сек.
25. **Зміна «users per page»** — при зміні значення в dropdown зависання ~1.8 сек.
26. **Пагінація Next/Last** — при переході на наступну сторінку (Next або Last) зависання ~1.2 сек.
